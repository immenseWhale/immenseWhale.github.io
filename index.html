프로젝트 개요 개발내용
제목 : 전자결재(Spring MVC) - 배포중
기간 : 2023-02-15 ~ 2023-03-27
인원 : 5명

반 전체가 참여한 파이널 팀 프로젝트입니다.
한 팀 당 6~7명이 큰 분류(플랫폼, 쇼핑몰, 예약시스템, 광고)를 맡게 되고,
한 팀 내에서도 업체측 담당과 고객측 담당으로 인원을 반으로 나누어 작업
* 나누어진 분류들은 각각의 war로 작업 (ex 예약 업체, 예약 고객…등)
 - 네이버를 참고하여 예약시스템 구현
- 업체측은 업체정보와 상품정보를 등록
- 등록과정에서 파일 업로드, 휴무일 설정, 이메일 인증, 지도 표시 등 기능 포함
- 예약 상태 관리, 고객 조회 가능
- 방문완료 예약 건에 한해 매출 통계 그래프 구현
담당기능
담당기능 : 로그인, 업체가입, 아이디/ 비밀번호찾기, 업체 기본정보, 전체/매출/신고통계

  1. 로그인
    • 로그인 분기
      - 계정 정보로 DB에 접근하여 로그인 분기를 나누었습니다.
    • 쿠키 사용하여 입력 정보 저장
      - 자바스크립트로 체크박스 활성화 여부 확인 후, 활성화 시 7일 동안 input에 입력된 정보를 쿠키에 저장하였습니다.
  2. 업체 기본정보
    • 동적버튼 활용한 파일 업로드
      - 업체 가입 시, 자바스크립트를 활용해 동적으로 사진 업로드 버튼이 생성되도록 구현했습니다.
      - multipart형식의 form으로 File타입의 객체를 받아와서 DB와 로컬폴더에 파일이 저장되도록 구현했습니다.
    • 구글 SMTP 서버와 JavaMail API 사용한 이메일 인증
      - 랜덤으로 생성된 인증번호와 API를 사용해 생성된 이메일을 구글 SMTP 서버를 통해 전송하였습니다.
      - 이메일의 전송과정은 비동기 방식으로 구현하였습니다.
    • 다음 지도 API 활용
      - 업체 가입 시, 사용자가 주소를 입력하면 주소 정보에서 위도, 경도를 가져와 DB에 저장하였습니다.
      - 업체 정보 조회 시, 저장된 위도와 경도 값으로 해당 위치가 지도에 표시되도록 구현하였습니다.
  3. 아이디/비번 찾기
    • 구글 SMTP 서버와 JavaMail API 사용한 이메일 인증
      - 아이디와 비밀번호를 찾는 과정에서도 비동기 방식으로 이메일을 전송하였습니다.
      - 입력한 인증번호가 일치하면, 비동기 방식을 통해 아이디 정보/비밀번호 변경 페이지가 나오도록 동적으로 구현하였습니다.
    • MyBatis 활용한 SQL 동적쿼리 사용
      - 인증을 위한 이메일 조회 시, 동적쿼리를 사용하여 하나의 이메일 전송 컨트롤러에서 세가지 방식의 이메일을 전송하도록 구현하였습니다.
  4. 통계
    • Chart.js 사용하여 매출통계 구현
      - 뷰에 차트 정보를 표시하기 위해 비동기 방식으로 Map타입의 List를 받아온 후, 배열로 변환하여 구현하였습니다.

mvc패턴 프로젝트

Cash 2023-07-10 ~ 2023-07-21

프로젝트 개요 개발내용
제목 : [MVC 패턴 프로젝트] : Cash
기간 : 2023-07-10 ~ 2023-07-21
인원 : 1명
 - 모델,컨트롤러,뷰로 나누어 제작하는 MVC 패턴 사용
- 관리자와 학생, 강사 기능 분리
- 관리자 - 관리자, 학생, 강사 삭제 및 등록 가능
- 강사 - 시험 출제 가능
- 학생 - 시험 조회, 답안 등록 가능
담당기능
담당기능 : 사원-사원/강사/학생 등록 및 조회, 강사-시험 등록, 학생-시험 조회, 점수확인

  1. 사원,강사,학생 등록
    • Ajax 활용한 아이디 중복검사 구현
      - 계정 등록 시, 비동기 방식으로 학생, 강사, 사원 테이블을 조회하여 아이디 중복검사 기능을 구현했습니다.
  2. 강사 시험 등록
    • MyBatis의 SelectKey를 활용하여 시험 문제 등록
      - 시험 등록은 시험 회차 → 해당 회차 문제리스트 → 해당 문제 보기리스트의 과정을 거쳐 등록됩니다.
      - ’A’시험 회차에 문제 등록 시, 1개의 문제와 4개의 보기가 동시에 등록되도록 구현했습니다.
      - 문제가 DB에 먼저 insert 되고, insert된 행의 key를 가져와 반복문을 통해 보기를 insert 하였습니다.
  3. 학생 시험 조회 및 답안 등록
    • 두 개의 리스트를 이중 반복문을 통해 출력
      - 문제리스트와 보기리스트를 각각 가져와 반복문을 통해 시험지를 구현했습니다.
      - 답안을 등록하는 테이블인paper에 데이터가 있는지 확인 후, 시험을 이미 응시한 학생은 점수 확인 페이지로 이동하도록 구현했습니다.
  4. 학생 점수 확인
    • 응시한 시험의 정답과 오답을 통해 점수 계산
      - 문제리스트와 보기리스트를 각각 가져와 반복문을 통해 페이지를 구현했습니다.
      - 정답과 오답을 반복문으로 비교하여 점수가 계산 되도록 구현했습니다.

- 관리자는 상품, 주문상태, 공지, 문의, 리뷰관리, 고객리스트조회 가능

담당기능
담당기능 : 쇼핑몰 메인, 상품 상세페이지, 리뷰 관리, 주문(바로 구매), 주문 관리, 리뷰 관리, 주소 관리

  1. 상품 상세페이지
    • 해당 상품에 해당하는 리뷰 조회 구현
      - 상품 테이블의 기본키를 이용해 상품 정보와 동시에 상품에 작성된 리뷰 리스트가 조회 되도록 구현했습니다.
    • 고객과 관리자를 구분하여 상품 구매
      - Controller에서 받아온 정보로 고객일 경우, 장바구니 및 바로구매 활성화, 관리자일 경우 비활성화 되도록 구현했습니다.
  2. 상품 주문(바로구매)
    • 주소 선택에 따른 분기
      - 주소 옵션 선택에 따라 주소 입력 분기를 나누었습니다.
    • 포인트 적용
      - 고객 포인트 정보를 가져와 포인트를 사용하면 주문금액이 변경되도록 구현했습니다.
  3. 주문관리
    • 한 jsp내에서 관리자, 고객 기능 구분
      - Controller에서 받아온 정보로 하나의 jsp에서 고객과 관리자가 보는 뷰와 기능이 다르게 작동하도록 구현했습니다.
