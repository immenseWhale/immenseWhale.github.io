위*아이텍 현업 참여 멘토링 		　이 프로젝트를 통해 스프링의 다양한 기능과 MVC 패턴, 서비스 레이어, preHandler를 실제로 적용해볼 기회를 얻었습니다.
　웹소켓과 STOMP헤더를 처음 접하며 채팅 기능 구현에 도전했습니다. 초기의 어려움을 겪었지만, 문제를 극복하고 구현을 완료하였습니다.
　기존에는 서버에서만 디버깅을 진행했지만, 웹소켓의 특성 상 클라이언트와 서버의 동시 디버깅이 필요했습니다. 이를 통해 개발 방식에 대한 새로운 인사이트를 얻었습니다. 프로젝트 완료 후, 팀원들과 함께 실제로 채팅 기능을 사용해보며 느낀 보람과 성취감은 잊지 못할 추억이 되었습니다.

문제상황: 웹소켓을 활용한 실시간 유저 목록이 제대로 동작하지 않는 문제 발생.
고민과 해결책: 웹소켓 세션과 싱글톤 패턴을 활용하여 문제를 해결. 초기 로딩 문제는 AJAX로 해결.
결과 : 실시간으로 유저 목록이 정확히 업데이트 되며, 모든 브라우저에서 일관된 동작을 보임.

제목 : 3차 개인 프로젝트 [MVC 패턴] Cash
기간 : 2023-07-10 ~ 2023-07-21
인원 : 1명
 MVC 패턴에 익숙해지는 프로젝트였습니다. MVC 패턴을 통해 각 구성요소의 역할을 분리하게 되어 프로젝트의 유지보수와 확장성이 크게 향상되었다는 것을 체감하였습니다.

사용자 입력 중 해시태그를 자동 변환하는 기능을 구현하면서 사용자 친화적인 인터페이스의 중요성을 더욱 깊게 이해할 수 있었습니다.
DB 쿼리에 GROUP_CONCAT을 활용하여 태그 출력의 효율성을 크게 높였습니다. 이 과정에서 데이터 처리 방법의 효율성에 대한 인사이트를 얻을 수 있었습니다.

이 달의 태그를 시각적으로 보여주기 위해 차트 API를 활용해 보았습니다. 이를 통해 외부 라이브러리의 프로젝트 통합 방법에 대해 학습하였습니다.

    • 비밀번호 확인

    • 로그인 인사
      - 첫 로그인시에만 환영 인사 팝업
  2. ﻿해시태그 생성과 GROUP_CONCAT을 활용한 출력

      - ﻿memo 입력 성공시 해시태그가 있는지 살피고, 해시태그가 있는 경우 해시태그에서 띄어쓰기까지 #을 떼고 태그로 저장
      - ﻿group_concat을 사용하여 LEFT JOIN을 깔끔하게 출력
  3. ﻿해시태그 생성과 GROUP_CONCAT을 활용한 출력

      - ﻿memo 입력 성공시 해시태그가 있는지 살피고, 해시태그가 있는 경우 해시태그에서 띄어쓰기까지 #을 떼고 태그로 저장
    • ﻿﻿group_concat 사용
      - ﻿﻿group_concat을 사용하여 해시태그와 가계부의 LEFT JOIN을 깔끔하게 출력

조명가게 쇼핑몰 2023-05-30 ~ 2023-06-23

프로젝트 개요 배운 점
제목 : 조명가게 쇼핑몰(MODEL1) - 배포중
기간 : 2023년 05월 30일 ~ 2023년 06월 23일 약 3주
인원 : 4명
 이 프로젝트를 통해 소통과 분배의 중요성을 크게 느꼈습니다.

이 프로젝트는 제 첫 팀 프로젝트였고, 네 명이서 역할을 분담하는 과정에서 예상치 못한 어려움들이 있었습니다.
저희 팀은 기획 작업을 통해 문서 작성 및 기획에 대한 이해를 높였습니다. 그러나 이 과정에서 필요한 추가 기능들이 많이 나타났습니다.
이를 통해 처음부터 완벽한 기획은 어렵다는 것을 깨달았고, 추가 기능에 대한 일정을 미리 고려해야 함을 인지했습니다.

특히, 제가 담당한 결제 시스템에서는 변수가 많이 발생했고, 이로 인해 추가 기능이 계속해서 추가되었습니다.
이러한 문제를 팀원들과의 소통을 통해 해결할 수 있었습니다.

저는 미리 역할 분배를 잘 조정하여, 팀원들이 저의 담당 파트 중 일부를 맡을 수 있도록 했습니다.
이로 인해 팀워크가 잘 이루어졌고, 함께 문제를 해결하면서 즐거운 경험을 할 수 있었습니다.

이 프로젝트를 통해 AJAX와 비동기 방식에 대한 흥미를 느꼈습니다.
특히 AJAX를 이용한 데이터베이스의 비동기 처리 방식이 매우 흥미로웠습니다.
이 경험을 통해, 저의 인지적 능력과 자기 파악 능력이 팀 프로젝트에 긍정적인 영향을 미칠 것이라고 자신합니다.
이는 앞으로 회사에서도 신입 개발자로서 중요한 덕목이 될 것이라 생각합니다.
담당기능
주문서와 결제, 주문 상세 페이지 등 주문 로직과 리뷰 CRUD

  1. 동적 주문 처리와 자동 배송 적용
    • DOM API를 활용하여 주문서에서 수량 변경에 따라 페이지 동적 처리
      - 주문서에서 상품 금액에 따라 배송비가 자동으로 무료배송 적용
      - 오늘의 날짜와 할인 기간을 비교하여 할인율 적용
      - 회원이 포인트를 적용시키면 원 금액과 사용한 포인트, 변경 된 최종 금액 동적 출력
      - 회원 포인트를 사용할 수 있지만, 배송비에는 포인트 적용 블가
    • 상품 주문 관리
      - 상품 페이지와 장바구니에서 모두 결제페이지로 주문을 요청 가능하도록 배열로 처리
      - null이 아닌 경우 request get parameter values를 사용하여 배열을 받아오고, 단일값과 배열을 모두 가능하게 처리
    • 다중 상품 주문 처리
      - 여러 상품을 여러 수량으로 한 번에 주문하기 위해 order와 product의 m:n(다대다) 관계를 교차 엔티티인 orderProduct로 설계
  2. 리뷰게시판
    • 이미지 업로드, 출력
      - 리뷰게시판에서 멀티파트(cos.jar)를 이용하여 이미지 업로드를 구현하고 출력
      - 리뷰에서 리뷰한 상품으로 이동 가능
    • DB JOIN 활용
      - 반정규화를 되도록 하지 않기 위해, 많은 조인을 사용. review-orderproduct-order-member 로 이어지는 테이블들을 조인하고 구체적인 where 사용으로 부하 감소
      - 많은 조인을 하게 되니 쿼리의 가독성의 중요성을 깨달았습니다. 한 줄로 쭉 이어쓰던 쿼리를 명령문에 맞춰 엔터와 들여쓰기로 구분해주고 쉼표를 앞에 쓰는 방식으로 혹시라도 놓칠 수 있는 쉼표가 강조되게 변경
